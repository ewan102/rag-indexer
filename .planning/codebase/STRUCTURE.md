## Naming Conventions

**Files:**

- `consumer.py`: Primary module (main async consumer, no package nesting)
- `producer.py`: Utility script in `scripts/` subdirectory
- `test_consumer.py`: Test module following pytest convention (test_*.py)
- `pyproject.toml`: Standard UV/pip project manifest
- `.env`: Standard dotenv convention for environment variables
- `.gitignore`: Standard git exclusion file

**Directories:**

- `scripts/`: Executables and utilities (not importable modules)
- `tests/`: Test modules parallel to implementation
- `data/`: Non-code assets (examples, fixtures, sample data)
- `.planning/`: Generated planning documentation

**Functions:**

- Snake_case: All functions use snake_case (process_message, rag_get_file, declare_topology)
- Async functions: Prefixed with `async def`, await on calls (lines 175, 332, 70)
- Helpers: Private scope implied by context, no leading underscore (consume is public via loop)
- Verbs: Start with action verb (process_message, declare_topology, build_metadata, get_producer_file, republish_to_retry, next_retry_routing_key)

**Variables:**

- Environment variables: UPPER_CASE with underscores (RABBITMQ_URL, EXCHANGE_NAME, MAX_RETRIES)
- Local variables: lowercase with underscores (retry_count, msg, session, headers)
- Constants: UPPER_CASE for configuration values read from env (line 15-27)
- Message fields: Snake_case attributes on Pydantic models (file_id, rag_base_url, content_type)

**Types:**

- Pydantic models: PascalCase (IndexMessage, RagConn, ContentSpec)
- Type hints: Present on function signatures (lines 70-76, 88, 113-118, 175-177)
- Dict keys: Match message header structure (lowercase with underscores, e.g., "rag_api_key")
